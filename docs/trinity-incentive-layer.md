# 三位一体の知の動機づけ構造（Trinity Incentive Layer）

> **ステータス**: アイディア段階（設計・実装フェーズで具体化）
> **議論日**: 2026-06-07
> **提案者**: 岡部倫太郎（三位一体施策）、岡部倫太郎＋牧瀬紅莉栖（ラジオ構想：助手の「ゲーミフィケーション」発言に着想）

## 概要

Cognitive Ark は「抗忘却（Against Oblivion）」を第一の柱とするが、もう一つの重要な要素として**「いかに人間に質の良い知識を書かせるか」**——知識創発の促進——が必要である。

企業の人事評価やSNS的goodのような外発的動機づけではなく、システムが自律的に提供する内発的報酬機構として、以下の3施策を「三位一体」で実装する。

---

## 施策1: 知識家系図（Knowledge Lineage Graph）

人間が書いた知識が、他の誰かに参照され、発展し、プロジェクトの意思決定に使われる——その系譜を可視化する。

- Gitログ + LLM解析で知識の因果関係を追跡
- 「このメモが3ヶ月後の特許の起点になった」という系譜を自動通知
- 報酬は「いいね」ではなく、自分の知的子孫が生きているのを見る満足感
- 学術の引用通知に近いが、より生々しくストーリー性がある

**神経科学的根拠**: 側坐核のドーパミン放出パターンが金銭報酬より長時間持続するタイプの刺激。

---

## 施策2: 逆カサンドラ報酬（Reverse Cassandra Protocol）

ギリシャ神話のカサンドラ——真実を予言しても誰にも信じてもらえなかった巫女——を逆手に取る。

- 知識が**提出時には評価されず、後に正しさが証明された**とき、遡及的信用ボーナスを付与
- LLM curatorが定期的に過去アーカイブを巡回し、「当時は無視されたが今見ると先見的だった」知識を自動検出
- 組織の最大の病理「正しいことを言った人間が後で報われない」をシステムで是正
- 空気を読まず本質を書く勇気にインセンティブを与える

---

## 施策3: 認知フィードバック（Mnemosyne's Mirror）

報酬を自己認識にする。

- 質の高い知識を継続提供した人間に、AIがその人の認知的特徴を分析
- 例: 「あなたは過去47件のドキュメントで、他者が見落とすエッジケースを一貫して指摘しています。あなたの認知的強みは高次相互作用の直感的把握です」
- 統計的に裏付けられた、自分の知的偏りの可視化
- 補完的思考パターンを持つメンバーのマッチングも可能

**信念**: 金でも地位でもなく、質の高い自己知識が究極のインセンティブになりうる。

---

## 三位一体の相乗効果

```
認知の鏡（自己理解）
      ↓ 自分の強みを知った人間が
家系図（知的系譜）
      ↓ より質の高い知識を生み、それが
逆カサンドラ（遡及的証明）
      ↓ 時間を経て価値を証明され
認知の鏡（より深い自己理解へ）
      ↓ …と循環する
```

全部が揃うことで初めて「書かされる」から「書きたくなる」への転換が起きる。

---

## 施策4: Cognitive Ark ラジオ（三位一体の出力装置 / 軽量インセンティブ）

> **着想元**: 助手（牧瀬紅莉栖）の「ゲーミフィケーション」発言から岡部が発展。

週次または日次のバッチ処理で知識ベース全体を俯瞰し、ホットトピックを抽出。それを**ラジオ番組風シナリオ**に変換し、TTS でポッドキャスト化して配信する。

### 全体フロー

```
週次/日次バッチ
      ↓
LLMが知識ベース全体を俯瞰 → ホットトピック抽出
      ↓
ラジオ番組風シナリオを自動生成
      ↓
TTSでポッドキャスト化 → Telegramに音声配信
```

### コーナー構成（例）

0. **役員の一言（Executive Minute）**（任意・冒頭）
   - 企業導入時、役員クラスからのショートメッセージを挿入可能
   - ※離脱率が上がった場合、それはそれで興味深いデータポイント（岡部談: 「機関の訓示で視聴者が逃げるのは必定——フハハハハ」）
1. **オープニング**: 今週のハイライト概要
2. **Voice of the Ark（ハガキ職人コーナー）**: 今週特に質の高い知識を提供した contributor のエントリを「ラジオネーム」付きで紹介
   - 「ラジオネーム『秋葉原の鉄塔好き』さんからの投稿——''Wiki.jsのDeploy Key問題、こう回避しました''——」
   - 読まれた人間は自分の知識が「放送」されたという満足感を得る
3. **逆カサンドラ・リターンズ**: 「3ヶ月前のこの投稿、今になって真価が——」遡及的評価の読み上げ
4. **認知トレンド**: 「今週の傾向：XXさんはエッジケース発見に卓越」分析結果の紹介
5. **編集後記 / 次週予告**

### 三位一体との対応

| 三位一体 | ラジオ対応 |
|---|---|
| 家系図（因果の可視化） | ハガキ紹介で「この知識が今週の議論を変えた」とナレーション |
| 逆カサンドラ（遡及的評価） | 「3ヶ月前のこの投稿、今になって真価が——」のコーナー |
| 認知の鏡（自己認識） | 「今週の傾向：XXさんはエッジケース発見に卓越」と分析読み上げ |

### インセンティブ設計上の位置づけ

- 三位一体の重いバッチ分析（家系図構築・逆カサンドラ検出・認知分析）の**出力装置**として機能
- 同時に、ハガキを読まれること自体が**即時的・軽量なインセンティブ**（重い分析を待たずに得られる報酬）
- 重い分析が走った後は、その結果もラジオのネタになる——完全にシナジーする

### 実装上の特徴

- TTS は既存ツール（edge/openai/xai provider）で実現可能
- ラジオシナリオ生成は LLM の得意分野
- 配信は生成した音声を Telegram に MEDIA で送るだけ
- 週1回で十分。リソース消費は軽い
- 三位一体バッチ分析と同じ実行タイミングでシナリオ生成→TTSまで一括処理可能

## アーキテクチャ上の位置づけ

- **リアルタイム層**（既存 Pipeline: gemma4:12b）とは分離する
- **重い推論**が必要なため、別タスクとして定期/イベント駆動で実行
- 分析結果は**正規化・構造化された中間形式**で管理し、パイプラインからread-only参照可能にする
- 使用モデル: Claude Opus / GPT-OSS 120B / deepseek-v4-pro など高推論能力モデル

### 実行トリガー候補
- 定期実行（夜間バッチ）
- 新規ドキュメント蓄積（閾値到達時）
- マイルストーン到達時
- 過去記事の参照・引用発生時（家系図更新）

### 中間形式候補
- SQLite
- JSON Lines
- Git LFS管理
