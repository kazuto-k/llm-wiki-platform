## 検証ポイント

| # | 検証項目 | 同一ホストでは検証不可だった理由 |
|---|---|---|
| 1 | Git が唯一の共有経路であること | ローカルファイル共有でバイパス可能 |
| 2 | 別ユーザのcommitが区別されること | 同一マシンだとGit設定が同じになりがち |
| 3 | ネットワーク越しのcurator応答時間 | 同一マシン内で完結してしまう |
| 4 | PR/MRベースのレビューフロー | 同一マシンなら直接編集で済んでしまう |
| 5 | 企業の部署別端末環境の再現 | 開発者が各自のマシンで作業する現実 |

## 各メンバーのCAへの貢献

| メンバー | ホスト | 役割 | CA蓄積コンテンツ |
|---|---|---|---|
| 岡部 | 本ホスト | 構想指揮 | 方針、アイディア、外部情報共有 |
| 助手 | 本ホスト | 分析評価 | 技術調査、ツール評価、設計検証、議事録 |
| ダル | MacBook | 実装技術 | 実装メモ、環境構築手順、ベンチマーク |
| まゆり | MacBook | 視点触媒 | ユーザ視点の気づき、直感的FB |

## 円卓会議フロー

```
Cognitive Ark から議題抽出
        ↓
Telegram グループ「未来ガジェット研究所・円卓会議」
  岡部＋助手＋ダル＋まゆり で議論
        ↓
助手が session_search で会話を取得・要約
        ↓
議事録を Cognitive Ark に蓄積（次の議題の種に）
```
